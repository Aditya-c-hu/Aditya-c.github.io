Rabin-Karp is a string matching algorithm that uses hashing to find a substring (pattern) within a larger string (text). It's efficient for searching multiple patterns and works well for metro system data processing.

A min-heap allows you to retrieve elements in sorted order incrementally, without needing to sort the entire dataset upfront. Example: If you only need the closest 3 metro stations out of 100, a min-heap can achieve this in O(3 log n) time, instead of sorting all 100 stations in O(n log n) time. This is well-suited for scenarios where data arrives dynamically, and you need to maintain a sorted order.
